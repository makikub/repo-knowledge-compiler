## Always Follow

- Before substantial work, check `.repo-kb/index.md` and relevant `.repo-kb/pages/`.
- During code review, use `REVIEW.md`, `.repo-kb/generated/review.md`, and relevant `.repo-kb/review-aspects/`.
- Propose new `.repo-kb` entries for repeated review findings, incidents, design decisions, and important repo-specific practices.
- Do not assume `CLAUDE.md`, `AGENTS.md`, or `REVIEW.md` are generated. Update `.repo-kb/`, run compile, then intentionally edit project guidance files if needed.
